Technical Documents(한국어) : 디스크 구성

Created by Shin beum-su on Mar 29, 2017

데이터베이스의 성능은 디스크 I/O 성능과 밀접한 관련이 있다. 기본적으로 검토되어야 하는 디스크 구성 관련사항에 대해 설명한다.

데이터베이스 전용 Storage

OS 나 응용프로그램 등과 같은 기타 외부적인 요소들의 디스크 I/O 로 인해 데이터베이스의 성능에 영향을 미치는 것을 원천적으로 배제해야 한다.  OS 및 응용프로그램 등의 기타 요소들이 사용하는 디스크와 물리적으로 분리된 데이터베이스만을 위한 전용 storage 가 기본적으로 요구된다

RAID

데이터베이스에 대한 RAID (Redundant Array of Independent Disk) 구성은 IOPS (Input Output Per Second) 가 가장 우수한 것으로 알려진 RAID 10이 주로 사용된다.  RAID 10 은 RAID 1 과 RAID 0 을 혼합한 방식이다.

RAID 구성방식에 따라서 사용되는 디스크의  수량이 달라질 수 있으므로  데이터베이스의 크기와 가용한 디스크 수량에 따라서  Storage 전문가와  협의하여 적정한 방식을 선택한다.

  • mirror (RAID 1)

동일한 데이터를 여러 디스크에 중복하여 저장하므로 데이터 손실에 유연하다

  • striping (RAID 0)

데이터를 블록 단위로 분할하여 여러 디스크에 분산해서 저장하므로 입출력 속도를 크게 향상시킬 수 있다

디스크 성능

디스크 성능은 시스템의 트래픽에 따라 달라지는 것으로 최대 트래픽 테스트를 통하여 산정한다.  특성상 Random I/O 가 빈번히 발생하는 디스크 DB 의 경우는 우수한 Random I/O 성능을 보이는 SSD 가 권장된다.
