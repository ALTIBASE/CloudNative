FAQ(English) : ERR-610CF The transaction table size of the replication does not match

Created by Education/Contents Team, last modified on Nov 20, 2014
  • Version
  • Explanation
  • Cause
  • Action
  • Reference

Version

4.3.9 or above

Explanation

Unable to start replication.

Cause

The TRANSACTION_TABLE_SIZE property for both local and remote servers is different.

Action

Set an identical value for TRANSACTION_TABLE_SIZE on both servers. Change it from the altibase.properties file located in $ALTIBASE_HOME/conf/ directory, then restart the server. Â 

Reference

1. TRANSACTION_TABLE_SIZE can be changed without recreating the database from Altibase versions 5.1.5.9.3 or above.

2. If TRANSACTION_TABLE_SIZE is changed, older versions need to restart the server.
